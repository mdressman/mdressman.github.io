    When can we use this?

    Now! Really? Well, the spec is "finalized" but we still need to polyfill while browsers catch up. Enter Picturefill

    Fortunately, Picturefill is being built to spec with support from the RICG so ideally it will be painless to rip out eventually.

    Note: issues for browsers without Javascript enabled. A bit more research needed on the Google scenario.

    Other Notes

    There are two primary use cases for responsive images.

    1. Serving the smallest / most optimal image to the device being used, based on viewport size and device pixel ratio. Described above. This could give us significant performance optimizations across the site.
    2. Art direction – use the <picture> element to define different media sources based on media queries.
      • A study found that less than a quarter of responsive image needs fall under the art direction use case. Not described here.
